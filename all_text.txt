I- Introduction
Ce projet a pour but de concevoir un mini-compilateur pour un langage nommé
«ProLang», en intégrant les différentes étapes de la compilation : l'analyse lexicale,
syntaxique, sémantique, la génération de code intermédiaire, optimisation du code
intermédiaire, ainsi que la génération du code objet.
De plus, la gestion parallèle de la table des symboles et le traitement des différentes erreurs
devront être pris en charge lors des phases d’analyse du processus de compilation.
II- Description du mini langage « ProLang»
II.1. Structure générale

II.2. Partie déclaration
Cette section est dédiée à la déclaration des variables simples et constantes nécessaires au bon
fonctionnement du programme.
1. Déclaration de variables
La syntaxe de déclaration d’une variable simple est la suivante :
define <nom_var> : <type> ;

2. Déclaration d’un tableau
define <nom_tab> : [<type> ;<Taille>] ;
 La taille du tableau doit être une constante entière positive
Exemple :
define Tab : [integer ;20] ;
3. Déclaration d’une constante
La syntaxe de déclaration d’une constante est la suivante :
const <nom_const> : <type>=Valeur ;
Exemple :
const Pi : float = 3.14159;
 Valeur est une constante.
3. Identificateur
Un identificateur est une suite de lettres minuscules, de chiffres et/ou de tirets bas (« _ »), qui
commence par une lettre alphabétique, ne dépasse pas 14 caractères, ne se termine pas par un
tiret bas et ne contient pas de tirets bas consécutifs.
 Les noms de programme, ainsi que les noms de variables et de constantes, sont des
identificateurs.
Exemple: Isil_program, Myprogram, testprog, ….

III- Travail à réaliser
Ci-dessous les différentes briques à implémenter afin de réaliser le compilateur demandé :
1. Analyse lexicale avec l'outil FLEX
Cette phase consiste à parcourir le programme source afin d’identifier les unités lexicales
(tokens) telles que les mots-clés, identificateurs, constantes, opérateurs et séparateurs.
2. Analyse syntaxico-sémantique avec l’outil BISON
L’analyse syntaxique vérifie la conformité du programme par rapport à la grammaire du
langage, tandis que l’analyse sémantique permet de détecter des erreurs liées au sens (types
incompatibles, identificateurs non déclarés, etc.).
3. Gestion de la table de symboles :
La table de symboles est créée lors de l’analyse lexicale et regroupe les informations
nécessaires sur les variables, constantes et structures du programme. Cette table doit être
implémentée sous forme d’une table de hachage et doit être mise à jour tout au long du
processus de compilation. Vous devez prévoir des procédures pour rechercher et insérer des
éléments dans la table des symboles, et inclure également les variables structurées.
4. Génération du code intermédiaire
Le code intermédiaire doit être généré sous forme de quadruplets.
5. Optimisation du code intermédiaire.
